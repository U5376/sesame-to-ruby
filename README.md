# EPUB傍点转Ruby工具
![图片描述](260312.avif)

## 项目概述
用于规格化处理日语EPUB的工具，主要功能是将em-sesame/em-dot样式的傍点转为Ruby标签格式。2.0开始堆了更多的东西用于自动化定制处理。

## 主要功能
- **傍点转Ruby**：将指定class的span/em标签转换为标准ruby标签
  - 支持自定义傍点class名称(默认em-sesame|em-dot)
  - 自动合并多个<rt>内容
  - 处理gaiji class的特殊图片文字
- **Ruby格式规格化**：处理格式奇怪和包含gaiji图片的ruby标签
  - 合并连续的ruby标签
  - 处理ruby内含图片的情况
  - 清理多余的rb标签
- **图片标签多看交互规格化**：将奇怪的图片标签规格化成多看格式
  - 处理div/p包裹的图片
  - 处理svg和ops:switch标签
  - 排除span跟gaiji标签(不处理P标签的,因为容易误伤)
- **正则预处理**：支持自定义正则规则清理HTML标签样式
  - 通过RegexManager管理正则规则
  - 支持规则保存和加载(config.ini)
  - 提供规则编辑的ToolTip提示
- **图片处理**：
  - 转换图片格式(支持webp/png/jpg)
  - 规格化图片标签(排除span跟gaiji标签)
  - 更新OPF清单中的媒体类型
- **章节合并**：
  - 根据目录结构合并XHTML文件
  - 支持排除特定章节
  - 分隔符样式可选(-/hr+br/1-8br)
- **正则追加、分割章节**：
  - 支持通过正则匹配追加 分割目录条目
  - 可选层级控制depth1-2(同级/子章节)
  - 预览功能可实时查看分割效果
- **生成ncx并更新opf**：自动生成和修正目录
  - 自动对照opf列表修正路径
  - 自动偏移功能(检测文件不存在时自动-1修正)
  - 强制偏移(手动设置偏移值，优先级高于自动)
  - 补全后记(自动补全ncx/nav缺失的あとがき条目)
- **删除自带Style并添加自定义样式表**：
  - 清理原有样式(item[media-type="text/css"])
  - 清理opf竖排属性(page-progression-direction)
  - 添加自定义CSS(style.css)
  - 规格化头部信息
- **转Epub2.0并删除nav.xhtml**：
  - 生成NCX目录(toc.ncx)
  - 移除nav.xhtml
  - 修改package声明为version="2.0"
- **语言标识**：
    opf跟head的头部语言标识参数
  - 默认ja.支持自定义填写zh-CN等语言标识
- **批量转换**：
    支持同时处理多个EPUB文件
  - 记录转换状态和错误
- **空行处理**：
    全局参数
  - 删除指定数量的空行
  - 限制连续空行的最大数量
- **class列表分析**：
  - 提取并统计HTML中使用的所有class
  - 支持文件树浏览、拖入拖出导出导入删除保存修改
  - 预览文件内容并支持正则匹配搜索
  - 样式详情查看(显示定义和实例)
  - 支持临时样式编辑和追加
  - 可按字母/总量排序
- **配置管理**：
  - 多个配置文件(ini)管理
  - 配置文件右键复制/删除/重命名
  - 窗口坐标大小记忆
  - 日志级别切换(debug/info)

## 文件结构
```
sesame-to-ruby/
├── sesame-to-ruby.py      # 主程序(GUI界面和主控制流程)
├── epub_ncx_generator.py  # ncx nav生成转换epub2.0与各种目录操作
├── regex_manager.py       # 正则规则管理(加载/保存/编辑/多配置)
├── class_list.py          # HTML class分析工具(文件树/拖拽导出导入保存修改/预览详情/搜索)
├── tooltip.py             # GUI元素悬浮提示
├── Image.py               # 图标资源(Base64编码)
├── image_converter.py     # 图片转换处理(nuitka编译为image_converter.exe)
├── style.css              # 自定义样式表
├── config.ini             # 配置文件(自动生成)
```

## 使用说明
1. **运行主程序**：`python sesame-to-ruby.py`
   - 初始化界面加载图标和配置
   - 自动加载上次保存的正则规则

2. **选择EPUB文件**：
   - 单文件处理：选择输入/输出路径
   - 批量转换：选择多个输入文件和输出目录
   - 支持拖拽epub文件到窗口

3. 根据需要配置转换选项：
   - **傍点转ruby**：设置需要转换的class名称，默认em-sesame|em-dot|kenten
   - **Ruby格式规格化**：处理格式奇怪和包含gaiji图片的ruby标签
   - **图片标签多看交互规格化**：将奇怪的图片标签规格化成多看格式，排除span跟gaiji
   - **Xhtml章节间合并**：根据目录合并章节间文件
     - 分隔符样式：-/hr+br/1-8br
     - 删除指定的空行数量
     - 限制连续空行的行数
   - **删除自带Style并添加自定义样式表**：清理原有CSS、opf竖排属性，添加style.css，规格化头部信息
   - **生成ncx并更新opf**：自动对照opf列表修正路径
     - 偏移：最后一条目录文件不存在时自动-1修正
     - 强制偏移：手动设置偏移值，0不执行
     - 补全后记：自动补全缺失的あとがき条目
   - **转Epub2.0并删除nav.xhtml**：转换为EPUB2.0格式，移除nav.xhtml
   - **转换图片**：设置图片转换参数
      -f webp -q80 -H1300 -W1200 -s1.0 -A -w
      (-f格式 -q质量 -H/-W高宽 -s锐化 -A透明 -w线程 -m压缩等级)
   - **语言标识**：设置opf跟head的语言参数，如ja、zh-CN
   - **正则规则**：通过界面编辑、添加、删除规则，右键编辑tooltip
   - **排除合并**：选择不合并的章节目录，支持正则追加分割章节预览

4. 开始转换并保存结果

## 执行流程
参考 逻辑流程 AI生成.txt
大致上
1. 解压EPUB到临时目录(sesame_cache)
2. 图片转换（可选）：转换图片格式、更新OPF媒体类型
3. 处理OPF文件和样式表
   - 删除自带Style并添加自定义样式表
   - 设置语言标识
4. 生成ncx并更新opf：自动对照opf列表修正路径
5. 修正ncx路径：目录偏移、强制偏移、补全后记
6. 转Epub2.0并删除nav.xhtml（可选）
7. 正则追加、分割章节（可选）：重新解析目录，正则匹配追加分割
8. 章节间合并（可选）：根据目录合并xhtml文件
9. 遍历所有xhtml/html文件处理：
   - Ruby格式规格化
   - 傍点转ruby
   - 正则替换
   - 图片标签多看交互规格化
10. 空行处理：删除指定空行、限制连续空行
11. 重新打包EPUB

## 注意事项
- 处理前请检查EPUB结构，过于奇葩的epub可能会出问题...目前检查转换处理数:394
- gaiji ruby内的图片文字 class名gaiji 会放出ruby标签前面
